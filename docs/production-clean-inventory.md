# Open Nova v1.0.0 Public Source Boundary

Status: current public-source and Runtime-payload contract.

Open Nova publishes product source, user documentation, and long-term
regression tests. It excludes private development history, internal audit
ledgers, one-time evidence harnesses, local Runtime data, dependencies, build
outputs, and credentials.

## Public source

| Path | Purpose |
| --- | --- |
| `config.py` | Checkout/runtime configuration bridge. |
| `pyproject.toml`, `MANIFEST.in` | Package and distribution metadata. |
| `install/` | Fresh bootstrap, installer, and guarded update transaction. |
| `advanced/cli/` | Product and compatibility CLI wrappers. |
| `advanced/dashboard/` | Dashboard and nova-RAG managed-service launch helpers. |
| `advanced/pipeline/` | Supported daily pipeline, Foundation refresh, Settings status, and Nova-Task reconciliation wrappers. |
| `src/data_foundation/` | Runtime paths, settings, database, migrations, pipeline, Scheduler, and Nova-Task authority. |
| `src/diary_generator/` | Narrative, technical, and learning generation passes. |
| `src/ai_assets_center/` | Source collection and usage attribution. |
| `src/dashboard/` | Dashboard API, services, and static UI. |
| `src/agentic_rag/` | Optional nova-RAG product source and benchmark data. |
| `tests/` | Long-term contributor and product-contract coverage with synthetic fixtures. |
| `docs/` | Current user, operator, architecture, release-page, and assurance documentation. |

Tests are public source assets. They are deliberately excluded from the
installed Runtime payload, wheel, and source distribution payload selected by
the installer.

## Runtime source payload

The Runtime source snapshot contains only:

- `advanced/` supported wrappers;
- `install/` bootstrap and transaction implementation;
- `src/` product packages, migrations, Dashboard static files, and RAG
  benchmark data;
- `config.py`, `LICENSE`, `MANIFEST.in`, and `pyproject.toml`.

It excludes README and documentation files, tests, Node dependencies, build
and distribution directories, package metadata generated by builds, caches,
bytecode, databases, logs, Runtime settings, environment files, and secrets.
The installer records the exact file set, per-file hashes, and aggregate hash
for each promoted Runtime source snapshot.

## Private and generated exclusions

| Excluded class | Reason |
| --- | --- |
| Private Git history and pre-v1 tags | The public project starts with the v1.0.0 root commit. |
| Internal audit, decision, planning, and evidence ledgers | Retained only in the private development archive. |
| Release-closeout, live credential-store, and machine-specific evidence harnesses | One-time audit tools are not long-term product contracts. |
| `.env` and `.env.*` except `.env.example` | Local secrets and machine-specific overrides. |
| Runtime databases, Settings, logs, and generated assets | User-owned state must never enter source or release artifacts. |
| `node_modules`, build output, test results, coverage, caches, bytecode, and egg-info | Reproducible local/generated content. |

## Runtime defaults

| Setting | Default |
| --- | --- |
| Runtime home | `~/.open-nova` |
| Active Runtime pointer | `~/.config/open-nova/location.json` |
| Generated diary output | `~/.open-nova/artifacts/diary` |
| CLI shim | `~/.open-nova/bin/open-nova`, optionally linked to `~/.local/bin/open-nova` |
| macOS Scheduler label prefix | `open-nova.daily.` |
| Dashboard LaunchAgent labels | `com.open-nova.dashboard`, `com.open-nova.dashboard.watchdog` |
