Open Nova

打通你的所有 AI 工具，自动为你总结工作，并沉淀专属于你的可检索、可复用 AI 资产。Open Nova 作为外挂式本地运行层，把分散在工具、CLI 和 agent 中的记忆、任务、经验与总结收拢进用户自己的 Foundation、Dashboard 与 Nova-RAG。

System Capabilities

从分散的 AI 工具活动，到自动总结、可检索的个人资产层。

Open Nova 不替代现有工具，而是外挂在它们旁边：采集受支持工具的活动，规范化为用户本地 Foundation 状态，并通过 Dashboard、Diary pipeline 与 Nova-RAG 自动形成工作总结、周期报告、任务证据和可复用记忆。

01

外挂式非侵入架构

读取配置好的工具位置，写入 Open Nova 自己的 runtime home，不接管用户的 agent、shell、编辑器或模型网关。

02

统一 AI 资产层

将工具、CLI、agents、sessions、usage events、tasks、reports 和生成日记规范化到用户本地 Foundation 边界中。

03

自动工作总结

Dashboard 与日记管线把分散 AI 活动自动整理为 daily summaries、period reports、technical progress、learning notes 和 task evidence。

04

专属 Nova-RAG

可选个人 RAG 子系统支持记忆检索、索引候选、评估、切换和回滚，并向外部 agent 提供只读检索合约。

05

Nova-Task 任务证据

任务 authority 与 review surface 将任务变更、证据和状态投影进本地看板，便于复盘跨工具 agent 活动。

06

安装与运维边界

bootstrap/install 流程支持 dry-run、运行时目录初始化、依赖分组、macOS 调度注册、doctor 检查和回滚视角。

Runtime Model

外挂在工具之外，资产沉淀在用户本地。

Open Nova 将采集、生成、物化、检索和展示拆分为可组合模块。默认 runtime 写入 ~/.open-nova，活动位置由 ~/.config/open-nova/location.json 指向；外部工具路径由 operator 显式配置。

1

Ingest

收集受支持 AI 工具、CLI、agent、workspace 与外部来源活动。

2

Pipeline

按业务日期自动生成日记、报告、学习与技术摘要。

3

Foundation

物化 SQLite 状态、快照、Markdown 和审计记录。

4

RAG

构建专属记忆索引，为 agent 和 operator 提供只读检索。

5

Dashboard

用本地 Web UI 汇总指标、任务、记忆、设置和运行状态。

发布与安全保证

  • 外挂式非侵入：不替代用户现有 AI 工具，只在旁侧收拢活动并沉淀资产。
  • 用户专属资产：记忆、任务、经验、总结、报表和检索索引都落在用户控制的本地 runtime。
  • 生产干净边界：不包含历史工作区日志、缓存、数据库、secret 或冻结脚本。
  • 不可变更新入口：one-liner 只做全新安装；更新默认从最新正式 Release 固定到完整 commit，绝不追踪 main/HEAD。
  • 可审计运维：Foundation repair、scheduler、settings 和 RAG skill registration 都有计划或记录边界。
  • 外部 agent 安全接口：RAG external namespace 只读，拒绝写入、索引、服务生命周期等 mutation。
